# Фаза 3 — План приведения к виду Nexus

Цель — **поднять качество в текущем стеке** (Node.js + `mysql2`, без TS-миграции и
без Postgres). Бот — отдельный проект со своей БД. План построен по фазам Nexus:
Стабилизация → Дисциплина границ → Контракт ошибок и данные → Тесты и CI →
Документация.

Оценки усилий: ч — часы, д — рабочие дни (один инженер).

---

## Фаза A — Стабилизация (чинить немедленно: баги/безопасность, дёшево)

| # | Инициатива | Проблема | Решение | Усилие | Definition of Done |
|---|-----------|----------|---------|--------|--------------------|
| A1 | Fail-fast по конфигу | B6, B7 | `assertConfig()` на старте `server.js`/`worker.js`: в production требовать `TELEGRAM_BOT_TOKEN`, `TELEGRAM_WEBHOOK_SECRET`, `ADMIN_PASSWORD_HASH`, креды БД; падать с понятным сообщением. Webhook без секрета — отклонять всегда. | ~3 ч | Старт без обязательной переменной завершается с ненулевым кодом и текстом; webhook без секрета → 403. |
| A2 | Атомарный захват задач | B1 | Claim перед отправкой (`UPDATE ... SET status='sending' ... LIMIT n` + выбор захваченных, либо `FOR UPDATE SKIP LOCKED`). `setInterval` → рекурсивный `setTimeout` после прохода. | ~1 д | Тест: два параллельных прохода не отправляют дубль; наложение проходов невозможно. |
| A3 | Обработка флуд-лимита | B2 | Разбор `retry_after`, ожидание+повтор с бэкоффом, лимит попыток; throttle между сообщениями рассылки; не ретраить 403 «blocked». | ~1 д | Рассылка на 200+ адресатов завершается без `failed` от 429; заблокировавшие — помечаются корректно. |
| A4 | Фильтр приватных чатов | B5 | В начале `handleMessage`/`handleCallback` игнорировать `chat.type !== 'private'`. | ~1 ч | Сообщения из групп не пишутся в `people` и не вызывают ответов. |
| A5 | Идемпотентность Facecast | B4 | Не вызывать `registerViewer`, если у регистрации уже есть `facecast_login`. | ~2 ч | Повторный тап/передоставка не создаёт нового зрителя и не меняет креды. |

**Итог фазы A:** ~3 дня. Снимает все High и часть Medium.

## Порядок и приоритеты

```
Чинить сейчас (баги/безопасность):     A1 → A2 → A3 → A4 → A5      (~3 д)
Привести к стандарту:                  B-1, B-2, B-3              (~3.5 д)
                                       C-1..C-4                   (~3 д)
                                       D-1..D-4                   (~5-6 д)
По мере развития:                      E-1..E-3, опц. TS-миграция
```

**Стартовать с дешёвых высокоэффективных:** A1 (fail-fast) и A4 (тип чата) — это
часы; A2/A3 — главный выигрыш по надёжности. Полная TS-миграция остаётся
опциональной инициативой «по мере развития» и не входит в обязательный объём при
выбранной цели.

## Definition of Done всего трека

- Все High и Medium-баги из [02-bug-report.md](02-bug-report.md) закрыты.
- Каждый внешний ввод валидируется; webhook защищён секретом и дедупом.
- Воркер не шлёт дублей и переживает флуд-лимит Telegram.
- Есть тесты (юнит + интеграция) и зелёный CI на PR.
- В корне есть `CLAUDE.md` и `RULES.md`.

# Аудит проекта Megapolis Event Bot

Результаты инженерного исследования бэкенда: реверс-инжиниринг, охота на баги,
gap-анализ против эталона Nexus и план приведения к нему.

**Дата:** 2026-06-18
**Аудируемая ревизия:** ветка `master`, коммит `a3bb942` (Add Docker Compose deployment)
**Зафиксированные решения заказчика:**

- Цель — **поднять качество в текущем стеке** (Node.js + `mysql2`), без миграции
  на TypeScript/Fastify/Prisma/Postgres.
- Бот — **отдельный проект со своей БД**, инфраструктуру с Nexus не делит.
  Следовательно, монорепо и переезд на Postgres из плана исключены.

## Вердикт одной строкой

Рабочий MVP с неожиданно дисциплинированной архитектурой (слои, параметризованный
SQL, экранирование, FSM в БД). Главные риски — не инъекции, а **надёжность
доставки** (дубли в воркере, отсутствие обработки флуд-лимита Telegram) и
**полное отсутствие тестов**. Это «поднять качество за 1–2 недели», а не
«переписать с нуля».
